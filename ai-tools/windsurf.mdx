Set up Windsurf's Cascade AI assistant to help you write and maintain your Mintlify documentation with intelligent, context-aware assistance.

## Prerequisites

- Windsurf editor installed
- A Mintlify documentation project

## Setup

<Steps>
<Step title="Open your docs directory in Windsurf">
  Open the root of your documentation repository where `docs.json` is located.
  
  ```bash
  cd your-docs-project
  windsurf .
  ```
</Step>

<Step title="Install the Mintlify skill">
  In the integrated terminal, run:

  ```bash
  npx skills add https://mintlify.com/docs
  ```

  This gives Windsurf Mintlify's component reference, writing standards,
  and workflow guidance.
  
  <Tip>
    The Mintlify skill teaches Cascade about MDX syntax, available components, and documentation best practices.
  </Tip>
</Step>

<Step title="Customize project instructions">
  Edit `AGENTS.md` in your project root to add project-specific terminology,
  style preferences, and content boundaries.
  
  Example customizations:
  
  ```markdown
  ## About this project
  
  - SaaS platform documentation
  - Audience: technical decision makers and developers
  - Tone: professional but approachable
  
  ## Terminology
  
  - "organization" not "company" or "account"
  - "integration" not "plugin" or "connector"
  - "dashboard" not "console" or "portal"
  
  ## Style preferences
  
  - Use sentence case for all headings
  - Include a use case for every feature
  - Code examples in TypeScript and Python only
  
  ## Content boundaries
  
  - Focus on production features only
  - Don't document beta features
  - Avoid implementation details
  ```
</Step>

<Step title="Start writing with Cascade">
  Open a file and use Cascade to draft and edit documentation.
  
  Access Cascade with:
  - Click the Cascade icon in the sidebar
  - Use keyboard shortcut `Cmd/Ctrl+Shift+C`
  - Right-click and select "Ask Cascade"
</Step>
</Steps>

## Example workflows

### Creating a tutorial page

Ask Cascade:

```
Create a tutorial page teaching users how to build their first workflow.
Use <Steps> to guide them through:
1. Creating a new workflow
2. Adding triggers
3. Configuring actions
4. Testing and deploying

Include code examples and screenshots placeholders.
```

### Adding code examples to existing pages

Select a section describing an API endpoint, then prompt:

```
Add a CodeGroup with examples in cURL, Python, and JavaScript
showing how to call this endpoint. Include authentication headers
and a sample response.
```

### Restructuring documentation

For larger reorganizations:

```
Restructure the 'Getting Started' section:
1. Move authentication content to a new auth/overview.mdx
2. Create a dedicated quickstart.mdx with a 5-minute guide
3. Update navigation in docs.json
4. Ensure all internal links still work
```

## Cascade features for documentation

### Flow mode

Cascade's Flow mode provides continuous, context-aware assistance as you write:

<Steps>
<Step title="Enable Flow mode">
  Click the Flow toggle in the Cascade panel or use `Cmd/Ctrl+Shift+F`.
</Step>

<Step title="Write naturally">
  Start typing your documentation. Cascade will offer inline completions that match your style.
</Step>

<Step title="Accept or modify">
  Press `Tab` to accept suggestions, or keep typing to ignore them.
</Step>
</Steps>

<Note>
  Flow mode learns from your `AGENTS.md` file and existing documentation to provide consistent suggestions.
</Note>

### Command mode

For specific tasks, use Command mode:

```
Cascade, add error handling examples to all API endpoint pages
under /api-reference. Include common HTTP status codes and
their meanings.
```

### Search and replace

Update terminology across your entire documentation:

```
Find all instances of "workspace" and replace with "organization"
in the /guides directory. Update AGENTS.md to reflect this change.
```

## Best practices

<Steps>
<Step title="Provide clear context">
  When asking Cascade to create content, specify:
  - Target audience and their experience level
  - Desired outcome (what should readers be able to do?)
  - Preferred components and structure
  - Any constraints (length, complexity, etc.)
</Step>

<Step title="Use the codebase as reference">
  Cascade can read your source code. Reference files directly:
  
  ```
  Document the authentication flow based on src/auth/middleware.ts.
  Create a page showing how developers implement it.
  ```
</Step>

<Step title="Review before committing">
  Always verify:
  - Technical accuracy
  - Code examples work correctly
  - Links are valid
  - Terminology matches `AGENTS.md`
  - Components render properly (run `mint dev`)
</Step>

<Step title="Iterate with feedback">
  Cascade responds well to refinement requests:
  
  ```
  Make this section more concise
  Add a tip about common gotchas
  Include a real-world example
  ```
</Step>
</Steps>

## Troubleshooting

<AccordionGroup>
<Accordion title="Cascade doesn't suggest Mintlify components">
  Ensure you've installed the Mintlify skill:
  
  ```bash
  npx skills add https://mintlify.com/docs
  ```
  
  Restart Windsurf after installation for changes to take effect.
</Accordion>

<Accordion title="Suggestions don't match our style">
  Update `AGENTS.md` with more specific examples:
  
  ```markdown
  ## Style preferences
  
  ✅ Preferred:
  Click **Settings** to open the configuration panel.
  
  ❌ Avoid:
  Click on the Settings button to open up the panel where you configure things.
  ```
  
  Cascade learns from these examples.
</Accordion>

<Accordion title="Flow mode is too aggressive">
  Adjust Flow sensitivity in Windsurf settings:
  - Open Settings > Cascade > Flow Mode
  - Lower the suggestion frequency
  - Or disable Flow and use Command mode for explicit requests
</Accordion>

<Accordion title="Cascade modifies the wrong files">
  Be explicit about file scope:
  
  ```
  Update only the files in /api-reference/users/ to include rate limit information.
  ```
</Accordion>
</AccordionGroup>
