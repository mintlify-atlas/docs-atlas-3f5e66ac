Reusable snippets help you maintain consistency and reduce duplication in your documentation. Any content you define once can be imported and used across multiple pages.

## The snippets directory

All snippet files must be created in the `snippets/` directory at the root of your docs project:

```
docs/
├── snippets/
│   ├── welcome-message.mdx
│   ├── api-auth.mdx
│   └── common-variables.mdx
├── essentials/
└── docs.json
```

<Note>
Any page in the `snippets/` directory is treated as a snippet and will **not** be rendered as a standalone page.
</Note>

## Creating snippets

### Default export (basic snippet)

Create a snippet file with content you want to reuse:

```mdx snippets/welcome-message.mdx
Welcome to our platform! We're excited to have you here. This guide will help
you get started with {productName}.
```

Import and use the snippet in your pages:

```mdx getting-started.mdx
---
title: Getting Started
description: Start using our platform
---

import WelcomeMessage from '/snippets/welcome-message.mdx';

<WelcomeMessage productName="Analytics Dashboard" />

## Next steps

Follow these steps to complete your setup...
```

<Tip>
Use curly braces `{variableName}` in snippets to create customizable content that accepts props.
</Tip>

### Named exports (variables)

Export variables from snippet files to reuse values:

```mdx snippets/product-info.mdx
export const productName = 'Mintlify Starter Kit';
export const version = '2.0';
export const supportEmail = 'support@example.com';

export const pricing = {
  starter: '$29/month',
  pro: '$99/month',
  enterprise: 'Contact us'
};
```

Import specific variables in your pages:

```mdx pricing.mdx
---
title: Pricing
description: Our pricing plans
---

import { productName, pricing } from '/snippets/product-info.mdx';

# {productName} Pricing

- **Starter**: {pricing.starter}
- **Pro**: {pricing.pro}
- **Enterprise**: {pricing.enterprise}
```

<Info>
Variables are perfect for maintaining consistent product names, versions, URLs, and other values that appear throughout your documentation.
</Info>

## Real-world examples

### Authentication snippet

Create a reusable authentication guide:

```mdx snippets/api-authentication.mdx
## Authentication

All API requests require authentication using an API key. Include your API key in the request header:

```bash
curl -X GET 'https://api.example.com/endpoint' \
  -H 'Authorization: Bearer YOUR_API_KEY'
```

You can generate API keys from your [dashboard settings](https://app.example.com/settings/api).
```

Use it in multiple API endpoint pages:

```mdx api-reference/users.mdx
---
title: Users API
---

import ApiAuth from '/snippets/api-authentication.mdx';

<ApiAuth />

## List users

Get a list of all users...
```

### Prerequisites snippet

Create a snippet for common prerequisites:

```mdx snippets/prerequisites.mdx
## Prerequisites

Before you begin, ensure you have:

- Node.js {nodeVersion} or higher installed
- A valid API key (get one from the [dashboard](https://app.example.com))
- Basic knowledge of {technology}

<Warning>
Make sure your API key has the appropriate permissions for this operation.
</Warning>
```

Use with custom values:

```mdx guides/api-integration.mdx
---
title: API Integration Guide
---

import Prerequisites from '/snippets/prerequisites.mdx';

<Prerequisites nodeVersion="18.0" technology="REST APIs" />

## Getting started

Follow these steps...
```

## Advanced patterns

### Reusable components

Create component functions in snippets:

```mdx snippets/custom-callout.mdx
export const CustomCallout = ({ type, title, children }) => (
  <div style={{ padding: '1rem', borderRadius: '0.5rem', border: '1px solid #e5e7eb' }}>
    <h4 style={{ margin: 0, marginBottom: '0.5rem' }}>{title}</h4>
    <div>{children}</div>
  </div>
);
```

<Warning>
MDX does not compile inside arrow function components. Use HTML syntax instead of MDX syntax, or use default exports if you need MDX features.
</Warning>

Import and use the component:

```mdx
import { CustomCallout } from '/snippets/custom-callout.mdx';

<CustomCallout type="info" title="Important Note">
  <p>This is important information for users.</p>
</CustomCallout>
```

### Conditional content

Create snippets that render different content based on props:

```mdx snippets/installation.mdx
export const InstallCommand = ({ packageManager = 'npm' }) => {
  const commands = {
    npm: 'npm install mintlify',
    yarn: 'yarn add mintlify',
    pnpm: 'pnpm add mintlify'
  };
  
  return (
    <div>
      <p>Install using {packageManager}:</p>
      <pre><code>{commands[packageManager]}</code></pre>
    </div>
  );
};
```

### Code examples

Share code examples across multiple pages:

```mdx snippets/api-example.mdx
<CodeGroup>

{"```"}javascript JavaScript
const response = await fetch('https://api.example.com/data', {
  headers: {
    'Authorization': `Bearer ${apiKey}`
  }
});
const data = await response.json();
{"```"}

{"```"}python Python
import requests

response = requests.get(
    'https://api.example.com/data',
    headers={'Authorization': f'Bearer {api_key}'}
)
data = response.json()
{"```"}

</CodeGroup>
```

Import in multiple API documentation pages:

```mdx
import ApiExample from '/snippets/api-example.mdx';

## Example request

<ApiExample />
```

## Troubleshooting

<Warning>
Snippets must be imported using absolute paths starting with `/snippets/`. Relative imports will not work.
</Warning>

### Import not found

Ensure the snippet file exists in the `snippets/` directory and the import path is correct:

```mdx
// ✅ Correct
import MySnippet from '/snippets/my-snippet.mdx';

// ❌ Incorrect
import MySnippet from '../snippets/my-snippet.mdx';
```

### Props not working

Make sure you're using curly braces in the snippet content:

```mdx
// ✅ Correct
Welcome to {productName}!

// ❌ Incorrect
Welcome to productName!
```

### Styling issues

Remember that arrow function components only support HTML, not MDX:

```mdx
// ✅ Correct (HTML)
export const MyComponent = ({ text }) => (
  <div><p>{text}</p></div>
);

// ❌ Incorrect (MDX in arrow function)
export const MyComponent = ({ text }) => (
  <Note>{text}</Note>
);
```
