title: 'Create Plant'
openapi: 'POST /plants'
---

Creates a new plant in the store. This endpoint allows you to add plants to your inventory with a name and optional tag.

## Request

### Body Parameters

<ParamField body="name" type="string" required>
  The name of the plant you want to add to the store. This field is required.
</ParamField>

<ParamField body="tag" type="string">
  Optional tag to categorize the plant (e.g., "succulent", "flowering", "fern", "tropical").
</ParamField>

### Request Body Example

```json
{
  "name": "Monstera Deliciosa",
  "tag": "tropical"
}
```

## Response

<ResponseField name="name" type="string" required>
  The name of the plant that was created
</ResponseField>

<ResponseField name="tag" type="string">
  The tag associated with the plant, if provided
</ResponseField>

### Success Response

```json 200
{
  "name": "Monstera Deliciosa",
  "tag": "tropical"
}
```

### Error Response

```json 400
{
  "error": 400,
  "message": "Invalid plant data provided"
}
```

```python Python
import requests
import json

url = 'http://sandbox.mintlify.com/plants'
headers = {
    'Authorization': 'Bearer YOUR_API_TOKEN',
    'Content-Type': 'application/json'
}
data = {
    'name': 'Monstera Deliciosa',
    'tag': 'tropical'
}

response = requests.post(url, headers=headers, data=json.dumps(data))
new_plant = response.json()
print(new_plant)
```

## Use Cases

- **Inventory management**: Add new plants to your store's inventory
- **User submissions**: Allow users to contribute new plants to the catalog
- **Bulk imports**: Create plants programmatically from external data sources
- **Product catalog**: Build and maintain your plant product offerings

## Validation Rules

- The `name` field is required and cannot be empty
- The `tag` field is optional but should be a valid string if provided
- Plant names should be unique to avoid duplicates (implementation-dependent)

## Best Practices

- Always validate plant data on the client side before sending the request
- Include descriptive tags to make plants easier to categorize and search
- Handle success and error responses appropriately
- Consider implementing duplicate detection before creating new plants
- Use meaningful plant names that help users identify the species
