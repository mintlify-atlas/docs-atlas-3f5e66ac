## Authentication

All API endpoints are authenticated using Bearer tokens and picked up from the specification file.

```json
"security": [
  {
    "bearerAuth": []
  }
]
```

## Setting Up Authentication

To authenticate your API requests, include your Bearer token in the Authorization header:

```bash
curl --request GET \
  --url http://sandbox.mintlify.com/plants \
  --header 'Authorization: Bearer YOUR_API_TOKEN'
```

```javascript
const response = await fetch('http://sandbox.mintlify.com/plants', {
  headers: {
    'Authorization': 'Bearer YOUR_API_TOKEN'
  }
});
```

```python
import requests

headers = {
    'Authorization': 'Bearer YOUR_API_TOKEN'
}

response = requests.get('http://sandbox.mintlify.com/plants', headers=headers)
```

## Base URL

All API requests should be made to:

```
http://sandbox.mintlify.com
```

## API Playground

Mintlify provides an interactive API playground for testing endpoints directly in the documentation. Each endpoint page includes:

- **Try it out**: Test the endpoint with your own parameters
- **Request builder**: Automatically generates code snippets in multiple languages
- **Response viewer**: See real-time responses from the API
- **Authentication**: Securely add your Bearer token to test authenticated requests

## Response Format

All responses are returned in JSON format. Successful responses will include the requested data, while errors will return an error object:

```json
{
  "error": 400,
  "message": "Error description"
}
```

## Rate Limiting

API rate limits are enforced to ensure fair usage. If you exceed the rate limit, you'll receive a `429 Too Many Requests` response.
